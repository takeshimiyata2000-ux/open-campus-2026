## 1. これは何か

- タンパク質の白いsurface（VDW表面）のくぼみに低分子をドラッグして置き、「ハマり具合」を0〜100点で採点するブラウザ完結型の教育ゲーム。
- ライブラリ・構造・低分子をすべてローカル同梱しているので、ネット接続なしでも（ローカルサーバー経由で）動く。
- GitHub Pagesで無料公開。オープンキャンパスで来場者（主に高校生）に触ってもらう想定。

## 3. アプリの仕組み（app.jsの要点）

### ステージ定義
`app.js` 冒頭の `STAGES` 配列にすべて集約。1ステージ＝1オブジェクトで、以下を追加するだけで新ステージを増やせる：
- `protein`（CIFパス）, `ligand`（SDFパス）, `proteinName` / `ligandName` / `blurb` / `mission`（表示テキスト）
- `timeLimit`（制限時間・秒）, `clearThreshold`（クリア点。ステージごとに変えられる）
- `answerKind`（「正解」の出し方。下記）と付随データ

現在の4ステージ：
1. **ラクトフェリン × カテキン**（食品ポリフェノール）clearThreshold 50
2. **ラクトフェリン × アンピシリン**（抗菌薬）clearThreshold 50
3. **コレラトキシン × GM1五糖**（毒素と宿主糖鎖）clearThreshold 35
4. **β-ラクトグロブリン × レチノール（ビタミンA）**（乳清タンパク質が脂溶性ビタミンを運ぶ）clearThreshold 50

### 採点 `scoreLigand()`
低分子とタンパク質の原子間距離から、
- **囲まれ方**（shellAtoms, 3.8〜8.2Å）… 最大45点
- **接触**（contacts, 2.2〜4.8Å）… 最大35点
- **衝突**（collisions <2.1Å の減点）… 最大20点
を合算（0〜100）。低分子の重原子数に応じて基準を自動スケール（大きい分子でも公平）。
- **`hasSolventAccess()`**: 低分子重心から「タンパク質中心と反対＝溶媒側」に抜け道があるか確認。抜け道がなければ（＝内部に完全に埋もれ）衝突が無くても0点。「表面のくぼみ」に正しく報いるための重要ロジック。閾値は暫定（到達距離 `ligandMaxRadius+6`Å、5Å以内に7原子以上で埋もれ判定）。

### 「正解」機能（answerKind）
- `"pocket"`（ステージ1,2）… カテキン/抗菌薬の本当の結合部位は未解明なので、**ゲームのルールで高得点になるくぼみ**を `computeGoodPockets()` で探索（初回クリック時に計算・キャッシュ）。マーカーは「お手本のくぼみ」。
- `"ctb"`（ステージ3）… コレラトキシンBサブユニット5鎖（TRP88を持つ鎖）のGM1結合残基から**実際の結合部位5か所**を算出し、毎回ランダムに1つ表示。
- `"hetsite"`（ステージ4）… 複合体構造に結合している低分子（HETATM, `hetCode`で指定）の位置＝**実際の結合ポケット**をそのまま正解に。BLGは `hetCode: "RTL"`（レチノール）。
- `seatLigandAt()` が正解サイトに低分子を最適姿勢で配置。近傍原子だけで採点して高速化。衝突を強めに減点し「埋まって見えない」きれいな姿勢を選ぶ。

### ヒントの3段階
- 「くぼみ候補を表示」（緑・複数候補、番号付き）
- 「ここだよ！（場所ヒント）」（赤・おすすめ1点だけ提示、低分子は動かさない）
- 「正解を見る（お手本）」（低分子を実際に配置）

### 低分子の回転（トラックボール）
- 姿勢は **回転行列 `state.pose.R`（3x3, 行優先の長さ9配列）** で表現（旧Euler角から変更済み）。
- 回転モードのドラッグは `applyTrackball()`。3Dmolのカメラクォータニオン（`viewer.getView()` の index 4〜7）から**画面基準の軸**を作り、低分子中心まわりに自由回転。固定軸の窮屈さを解消。
- お手本/ヒント探索は角度サンプルを `eulerToMatrix()` で行列に変換して使うので、採点挙動は不変。

### ランキング・名前登録・記録（localStorage）
- `pocketGame.records.v1` … 各ステージのセッション最高点（ステージボタンの「★NN」）。**リセットで消える**（次の来場者用）。
- `pocketGame.leaderboard.v1` … **ステージ別**の上位5件（消えない殿堂）。リセットでは消えない。旧形式（全ステージ混在配列）からの自動移行あり。
- 上位5件に入るクリア時のみ、**成功演出の約2.6秒後**に右パネル上部へ名前登録欄（アーケード風3文字イニシャル、▲▼選択）が出る。「名前なしで登録」も可。3Dビューには被らない。リセット/ステージ切替で保留はキャンセル。

## 4. データの出典（低分子・構造）

- タンパク質構造（CIF）: RCSB PDB（`https://files.rcsb.org/download/XXXX.cif`）。
  - 1blf=ウシラクトフェリン, 1xtc=コレラトキシン, 1gx8=BLG+レチノール。
- 低分子SDF:
  - PubChemの3D構造: アンピシリン(CID 6249), ciprofloxacin(2764), galactose(6036), lactose(6134), sialic acid(439197)。
  - **抽出系**（複合体PDBの結合リガンドの実座標＋原子間距離で結合再構成）:
    - `gm1_pentasaccharide.sdf` … PDB 3CHB（コレラトキシンB–GM1複合体）のGM1五糖。
    - `retinol.sdf` … PDB 1GX8のレチノール(RTL)。
  - 抽出スクリプトはPython（BioPython不要、CIF/SDFを直接パース）。同様の手順で新リガンドも作れる。

## 6. デプロイと認証（⚠️ここが自宅で唯一つまずくポイント）

- 公開はGitHub Pages（Settings→Pages→Source: `main` / `/ (root)`）。すでに有効化済み。
- 直下 `index.html` が `pocket_game/` へ自動リダイレクトするので、配布URLは `.../open-campus-2026/` だけでよい。
- **push認証**: 職場PCでは、このリポジトリ専用のSSHデプロイキー（`~/.ssh/id_ed25519_opencampus`）を作り、GitHubの Deploy keys に「write権限つき」で登録して使っている。リモートはSSH、`git config core.sshCommand` でこの鍵を指定している。
  - **この鍵は職場PCにしかない。** 自宅PCから push するには、自宅で別途認証を用意する必要がある。おすすめは次のどちらか：
    1. **GitHub CLI**: `brew install gh && gh auth login`（ブラウザ認証）→ リモートをHTTPSに戻す（`git remote set-url origin https://github.com/takeshimiyata2000-ux/open-campus-2026.git`）→ 普通に `git push`。
    2. **自宅用のデプロイキーを新規作成**: `ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_opencampus_home -N ""` → 公開鍵(.pub)をGitHubの Deploy keys に write権限つきで追加 → `git config core.sshCommand "ssh -i ~/.ssh/id_ed25519_opencampus_home -o IdentitiesOnly=yes"`。
  - 秘密鍵そのものをメール等で持ち出すのは避ける（新規に作る方が安全）。
- `.claude/`（launch.json等のローカル設定）は `.gitignore` 済みでリポジトリに入らない。自宅では必要なら作り直す（プレビュー用の些細な設定なので無くても可）。

## 7. 動作確認のやり方

- ローカルサーバー（`python3 -m http.server`）で開き、各ステージで「低分子移動→判定→クリア演出」まで一通り。

- Claude Codeで作業する場合は、プレビュー系ツールでローカルサーバーを立て、`app.js` を `node --check` で構文確認 → ブラウザで挙動確認、の流れが確実（実機描画の確認はAIからは限界があるので、最後は人の目で一度プレイ推奨）。
